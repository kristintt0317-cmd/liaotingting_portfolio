Generative Mechanical Score

An interactive visual-audio system that reinterprets everyday mechanical noise as a playable generative score. Mechanical recordings and synthetic sounds drive flowing staff lines, note objects, particles, scan overlays, and colour transitions in a Pygame environment.

Overview

Reframing machine noise as a visual audio agent.

This project is best understood as a hybrid of research note, creative coding experiment, and interactive artwork.

Starting point

The project begins with an everyday annoyance: hearing washing-machine noise repeatedly from a nearby laundry room. Instead of treating the sound as meaningless disturbance, the project asks whether it can be reinterpreted as rhythmic material for a playable visual score.

Design response

Five edited mechanical sound clips are combined with two synthetic sine waves and one white-noise clip. Their audio features are extracted, normalized, and mapped to visual behaviour so users can click notes on screen and experience each sound through motion, colour, and particle change.

Project story

From sound collection to generative interaction.

The project evolved by scaling ambition down in the right way: from mechanical composition to a more feasible sound-to-visual translation system.

Audio preparation

The system reads five edited mechanical recordings and generates three reference signals inside the code: two sine waves near musical frequencies and one white-noise clip. Stereo files are converted to mono, saved to CSV, and normalized before analysis.

Feature extraction

Each sound is described through mean absolute amplitude, normalized spectral centroid, and high-frequency ratio. These values become concise descriptors for loudness, brightness, and noise content, and later drive the agent's visual state.

System architecture

The Python program is divided into classes: an Agent stores current and target visual states, Line objects draw animated staff lines, Note objects act as clickable triggers, Particle objects add motion, and a Score class updates and renders the system as a whole.

Visual mapping

Amplitude controls curvature and brightness, jitter controls additional random movement, and spectral centroid maps frequency range to colour. Lower and duller sounds lean blue, while brighter noisy sounds shift toward pink and purple, so different clips feel visually distinct.

Iteration

The project originally aimed to generate more complex musical structures from mechanical sound, but that analysis proved too ambitious with the available methods. The concept then shifted toward a simplified yet stronger outcome: a fluid visual stave system that softens noise through interaction.

Reflection

The result is modest but complete: the system runs reliably, presents different sounds with clearly different behaviours, and opens room for future expansion through richer note layouts, real-time input, and more advanced mappings such as onset or envelope analysis.

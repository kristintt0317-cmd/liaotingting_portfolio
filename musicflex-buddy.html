Soft Robotics · Wearable Textile · Inclusive Music Interaction

MusicFlex Buddy

A music-interactive soft robotic system designed for users with upper-limb mobility limitations. The project uses knee movement as input, then maps flex-sensor data to pneumatic actuation and sound for rehabilitation, learning, and performance.

Overview

Translating body language into musical expression.

This page condenses the slide deck and written report into a web-first story focused on concept, system architecture, and technical debugging.

Design challenge

Traditional instruments depend on precise hand movements, which can create pain, access barriers, and limited expressive options for people with upper-limb disabilities. The project therefore explores whether larger body movements can become a more comfortable and emotionally meaningful input for music-making.

Design response

The team chose knee motion as the primary input. Flex sensors embedded in a wearable textile read knee bending, the system converts those values into pneumatic inflation and deflation, and pressure-based triggering produces rhythm and pitch through a speaker system.

Project story

From user selection to system debugging.

The key contribution on this project was debugging the pneumatic drive and improving sensor stability across the system.

Target user and concept

The project deliberately focuses on users with upper-limb disabilities rather than broader disability categories, arguing that many existing resources already concentrate on audio-visual impairment. Because most instruments rely on hands, the design opportunity is both practical and emotionally relevant.

System architecture

The system uses a knee sleeve with flex sensors as the input layer, Arduino UNO as the processing layer, and pneumatic bladders plus DFPlayer-driven sound as the output layer. The interaction path is clear: knee bending → flex value change → actuation logic → rhythmic sound and expanding form.

Material and form

Wearable textile anchors the sensors to the body, while lightweight plastic airbags, transparent tubing, acrylic supports, and speaker hardware create a leaf-like kinetic form. The structure makes sound visible, not just audible, which strengthens the performative quality of the system.

Technical development

Signal instability first appeared in the flex sensors due to grounding and resistor issues. Later, the pump-drive section was rewired from an H-bridge approach to a double-throw relay setup so the system could inflate and deflate more effectively with less current loss.

Calibration

The project then tuned flex thresholds, pump activation duration, and delay values to make stronger knee flexion produce quicker inflation and earlier pitch changes. Threshold testing on the FSR sensors reduced false triggering and made the mapping between motion, deformation, and sound more coherent.

Reflection

The concept works as a complete chain, but air-pump noise, response speed, wearable fit, and the durability of the airbag system still limit refinement. A future version would use quieter pneumatic components, stronger signal filtering, and a more resolved physical finish for rehabilitation and performance settings.
